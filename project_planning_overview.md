# 📘 Project Planning Overview – Study Notes

## 🧩 What is Project Planning?
Project planning is a core part of project management that involves organizing, defining, and structuring all the work needed to successfully complete a project.

---

## 🔍 Key Steps in Project Planning

### 1. Identify Work
- Determine all tasks required to complete the project.
- Define the full scope of work clearly.

### 2. Break Down Tasks
- Divide work into smaller, manageable pieces (*work breakdown structure*).
- Helps simplify execution and tracking.

### 3. Estimate Resources
- Assign:
  - **Who** will do the work
  - **Time** required for each task
  - **Cost** associated with tasks

---

## 📅 Scheduling the Project

Creating a project schedule involves:

- Understanding **task dependencies** (which tasks rely on others)
- Considering **resource availability**
- Accounting for **timelines and constraints**

✅ Requires structured techniques to ensure accuracy.

---

## 📢 Communication Planning

Define how the project will be managed:

- How the **team communicates**
- Which **tools** will be used
- How often updates will occur

---

## ⚠️ Supporting Plans

Additional elements included in planning:

- **Risk Management Plan** → Identify and mitigate risks  
- **Change Management Plan** → Handle project changes effectively  
- **Quality Plan** → Ensure deliverables meet standards  

---

## 📂 The Project Plan

All planning outputs are compiled into a **set of documents** known as the project plan.

### 📌 Important:
- The project plan is a **living document**
- It is used throughout the project to:
  - Guide execution
  - Track progress
  - Adjust direction when needed
  - Communicate with stakeholders

---

## 🏁 Key Takeaway

> A well-built project plan is not static—it actively drives the project from start to completion.
