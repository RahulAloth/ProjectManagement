# 📘 Study Notes: What is Agile Project Management?

## 🔹 1. Definition of Agile Project Management
- **Agile Project Management** is an approach that:
  - Delivers work in **small, usable pieces (features)**
  - Adapts quickly to **changing business needs**

✅ Best suited for:
- Projects where requirements change frequently
- Situations where **early value delivery** is important

---

## 🔹 2. Key Concept: Iterations (Sprints)
- Work is divided into **iterations (or sprints)**
- Typical duration:
  - ⏱️ 2 to 12 weeks

### Goal of each iteration:
- Deliver a **complete, ready-to-use feature**

✅ Result:
- Continuous delivery of value

---

## 🔹 3. Core Principles of Agile

### ✅ 3.1 Focus on Value Delivery
- Deliver working features **frequently**
- Provide value early and often

---

### ✅ 3.2 Embrace Change
- Changes are:
  - ✅ Expected
  - ✅ Welcomed
- New requirements can be added in future iterations

---

### ✅ 3.3 People Over Processes
- Emphasizes:
  - Collaboration
  - Communication
- Less reliance on:
  - Strict processes
  - Formal meetings

---

### ✅ 3.4 Customer Collaboration
- Customer is actively involved throughout the project
- Continuous feedback improves:
  - Product quality
  - Requirement alignment

---

### ✅ 3.5 Minimal Documentation
- Focus is on:
  - **Working product**, not excessive documentation
- Only essential documentation is created when needed

---

## 🔹 4. Agile vs Waterfall Approach

### Waterfall:
- Define **scope first**
- Then estimate:
  - Time
  - Cost
  - Quality

---

### Agile:
- Fix:
  - Time
  - Cost
  - Quality
- Then decide:
  - Which features can be delivered within constraints

---

## 🔹 5. Agile Lifecycle Stages

### ✅ 5.1 Envision
- Define:
  - Goals
  - Project boundaries

---

### ✅ 5.2 Iterative Cycle
Repeated phases:

#### Speculate
- Plan what features to build

#### Explore
- Develop and test features

#### Adapt
- Review and adjust based on feedback

---

### ✅ 5.3 Close
- Finalize project
- Wrap up activities

---

## 🔹 6. Managing Work in Agile

### ✅ Backlog
- List of features to be developed
- Continuously updated and prioritized

---

### ✅ Change Control
- Managed by:
  - Re-prioritizing the backlog
- No rigid change approval process

---

### ✅ Risk Management
- Reduced through:
  - Frequent deliveries
  - Continuous feedback

---

## 🔹 7. Agile Beyond IT
- Agile is **not limited to IT projects**

✅ Works well for:
- Any project where:
  - Deliverables can be broken into small parts
  - Work can be done incrementally

---

## 🔹 8. When to Use Agile
- Requirements are **uncertain or evolving**
- Need to deliver value **quickly**
- High customer involvement is required

---

## 🔹 ✅ Key Takeaways
- Agile delivers work in **small, usable increments**
- Uses **iterations (sprints)** for progress
- Encourages:
  - Change
  - Collaboration
  - Continuous improvement
- Focuses on **value over documentation**
- Ideal for dynamic and fast-changing projects

---
# 📘 Study Notes: Agile Project Management Lifecycle (Jim Highsmith)

## 🔹 1. Overview of Agile Lifecycle
- Agile lifecycle (by **Jim Highsmith**) consists of **5 stages**:
  1. Envision  
  2. Speculate  
  3. Explore  
  4. Adapt  
  5. Close  

✅ Key Idea:
- Instead of planning everything upfront, Agile:
  - Repeats cycles to **continuously build and improve features**

---

## 🔹 2. Envision Stage (Project Start)

### 🎯 Purpose:
- Define what the project will deliver

### Key Activities:
- Define:
  - Project goals and objectives
  - Scope and boundaries
  - Stakeholders
- Identify:
  - Project team
- Establish:
  - Communication and collaboration rules
  - Working guidelines (hours, processes, etc.)

✅ Output:
- **Project Definition Document**

---

## 🔹 3. Iterative Cycle (Core Agile Loop)

After envisioning, Agile follows a repeating cycle:

    Speculate → Explore → Adapt → Repeat

---

## 🔹 4. Speculate Stage (Iteration Planning)

### 🎯 Purpose:
- Plan the **next iteration (not the whole project)**

### Key Activities:
- Select features from backlog
- Define:
  - Business functions to deliver
- Estimate:
  - Effort for each feature
- Identify/update:
  - Risks

✅ Output:
- **Iteration plan**

---

## 🔹 5. Explore Stage (Development & Testing)

### 🎯 Purpose:
- Build and test features

### Key Activities:
- Develop features
- Conduct:
  - Frequent collaboration between teams
  - Peer reviews
- Perform testing:
  - Ensure features are **production-ready**

✅ Focus:
- Continuous communication
- Rapid feedback

---

## 🔹 6. Adapt Stage (Review & Improve)

### 🎯 Purpose:
- Evaluate and improve results

### Key Activities:
- Customer/Product Owner review
- Validate completed features
- Conduct **team retrospective**
  - Capture lessons learned
- Adjust:
  - Plans
  - Processes for next iteration

✅ Output:
- Improved approach for future iterations

---

## 🔹 7. Iteration Loop

- Repeat:
    Speculate → Explore → Adapt
- Continue until:
  - All features are complete OR
  - Time/budget is exhausted

---

## 🔹 8. Close Stage (Project Completion)

### 🎯 Purpose:
- Finalize the project

### Key Activities:
- Confirm:
  - All work is complete
- Document:
  - Final deliverables
  - Lessons learned

✅ Output:
- Project closure documentation

---

## 🔹 9. Agile vs Waterfall Lifecycle

### Agile:
- Iterative and flexible
- Continuous feedback
- Incremental delivery

### Waterfall:
- Sequential
- Fixed upfront planning
- Delivery at the end

---

        Speculate → Explore → Adapt
        ↓
    Iteration 2:
        Speculate → Explore → Adapt
        ↓
    Iteration 3:
        Speculate → Explore → Adapt
        ↓
    Iteration 4:
        Speculate → Explore → Adapt
        ↓
    [Close]

---

## 🔹 ✅ Key Takeaways
- Agile lifecycle has **5 stages**
- Work is delivered in **iterations**
- Focus is on:
  - Continuous improvement
  - Customer feedback
  - Incremental value delivery
- Iterative cycle allows:
  - Faster start
  - Better adaptability
- Lessons learned are captured continuously and at closure
