## Overview

Projects begin when **a problem needs solving** or **an opportunity is identified**.  
The ultimate goal of every project is to deliver value and benefits to the organization.

As a **project manager**, your role involves continuously balancing:
- **Project scope**
- **Time**
- **Resources**
- **Budget**

Success depends on managing these constraints effectively while keeping stakeholders aligned.

---

## Why Projects Matter

Organizations invest in projects to:
- Achieve strategic goals  
- Improve processes or products  
- Realize measurable business benefits  

Project managers, in turn, gain:
- Exposure to new industries  
- Opportunities to manage global initiatives  
- Continuous professional growth  

---

## Core Responsibilities of a Project Manager

A project manager must:
- Start projects **the right way**
- Build a **comprehensive project plan**
- Keep execution **on track from start to finish**
- Adjust proactively when risks or issues arise  

This role is an ongoing **balancing act** between expectations and constraints.

## Waterfall Project Lifecycle Focus

The course primarily follows the **Waterfall approach**, covering:

1. **Project Initiation**
   - Getting a project approved
   - Authorization to proceed

2. **Planning**
   - Building a project plan
   - Creating schedules using Waterfall techniques

3. **Agile Overview**
   - A short detour to understand Agile project management concepts

4. **Execution**
   - Communication practices
   - Running effective meetings
   - Working with teams during project execution

5. **Monitoring & Control**
   - Tracking project performance
   - Keeping the project on plan
   - Corrective actions to get back on track if needed

6. **Closing**
   - Formal project closeout steps in a Waterfall project
